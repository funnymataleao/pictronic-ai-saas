# JUP-153 — Волна дизайн-ревью (hero animation неизменяема)

## Ограничение (неизменяемое)
- `Hero animation` (sparkles beam, поведение частиц, тайминги анимации) является неизменяемой.
- Нельзя менять разметку и поведение внутри hero-блока в `components/public-landing.tsx`.
- Допускаются только правки отступов и типографики вокруг hero-блока без вмешательства в логику анимации.

## Findings
1. Слабая иерархия основного CTA на первом экране.
- На desktop видна только кнопка `Log in` в navbar, в hero-контенте нет основного conversion CTA.
- Влияние: неочевидный сценарий первого действия для новых пользователей, ниже конверсионная ясность.

2. Информационные секции визуально перегружены и однотипны.
- `Features`, `Workflow`, `Pricing`, `FAQ`, `Proof`, `Documentation`, `Terms` имеют близкий визуальный вес и похожую карточную подачу.
- Влияние: медленное сканирование страницы и слабая сюжетная последовательность.

3. Нестабильный вертикальный ритм между секциями.
- Сильный отрицательный offset от hero к feature-картам и неоднородные paddings создают визуальные скачки.
- Влияние: страдает ощущение системности и качества композиции.

4. Повышенная нагрузка на читаемость в mobile.
- `Pricing` и `FAQ` корректно уходят в одну колонку, но плотность текста остаётся высокой.
- Влияние: рост когнитивной нагрузки на экранах <=720px.

5. Доверительные сигналы удалены от точки первичной конверсии.
- Блок надёжности есть в `#proof`, но он расположен слишком далеко от первого действия пользователя.
- Влияние: ценность продукта и доказательство надёжности разнесены в потоке чтения.

## Required Fixes
1. Добавить основной CTA в hero-контент без изменения поведения hero animation.
- В `components/public-landing.tsx` добавить CTA-ряд под subtitle:
  - Primary: `Start in Workspace` (тот же auth flow, что у текущего login).
  - Secondary: якорь на `#proof` (`View runtime guardrails`).
- Текущую кнопку login в navbar сохранить.

2. Ввести уровни визуальной иерархии секций и убрать одинаковый вес блоков.
- Tier A: Hero + Features + Proof.
- Tier B: Workflow.
- Tier C: Pricing + FAQ.
- Tier D: Documentation + Terms + Footer.
- Реализовать через spacing/tokens в `app/globals.css`:
  - Интервалы (desktop): A->A `40px`, A->B `56px`, B->C `48px`, C->D `40px`.
  - Для mobile <=720px уменьшить интервалы примерно на 20%.

3. Нормализовать ритм секций и убрать резкий overlap-эффект.
- Ослабить отрицательный сдвиг feature-секции:
  - `.landing-value` margin-top с `-2.6rem` до управляемого значения (`-1.2rem` desktop, `0` mobile).
- Унифицировать внутренние отступы:
  - cards `16-20px`, section-shell `20-24px` desktop, `14-18px` mobile.

4. Повысить сканируемость mobile для pricing/faq.
- Ввести контроль длины строки и более явное дробление текста:
  - max ширина длинных описаний: `60-65ch` desktop, `100%` mobile.
  - увеличить межэлементные интервалы на `2-4px`, уменьшить размер paragraph-шрифта на `~0.03-0.05rem` только в зонах переполнения.
- Одноколоночный layout для <=1024 оставить.

5. Подтянуть trust-сигнал ближе к точке конверсии.
- Добавить компактную trust-strip сразу под hero CTA-рядом (2-3 тезиса):
  - `Guarded actions`
  - `Deterministic recovery`
  - `Private workspace`
- Использовать текущий визуальный язык, без добавления тяжёлой карточки.

## Optional Polish
1. Добавить мягкую группировку reveal-анимаций по tier-секциям для усиления повествовательного порядка.
2. Сократить формулировки FAQ-вопросов для быстрого сканирования (цель: <=72 символов на вопрос).
3. В pricing-описаниях заменить общие формулировки на outcome-oriented microcopy.

## Acceptance Criteria (Frontend)
1. Hero animation поведенчески идентична текущей версии.
- Не меняются particle-конфиг, поведение beam-линий и тайминги hero motion.

2. Первый экран имеет явный двухшаговый action-path.
- В hero-контенте есть primary CTA на desktop и mobile.
- Рядом с primary CTA есть secondary CTA, ведущий к trust-блоку.

3. Иерархия секций визуально считывается.
- По скриншот-сравнению Proof-блок визуально приоритезирован относительно Pricing/FAQ.
- Documentation/Terms визуально де-акцентированы как поддерживающий/legal-контент.

4. Mobile parity сохранена.
- На ширине 360px нет горизонтального скролла.
- Pricing и FAQ читаются без клиппинга и overflow.

5. Ритм секций проходит визуальный QA.
- Межсекционные интервалы следуют фиксированным токенам, без резких скачков между hero/features/workflow.

## Design Verdict
- Текущее состояние: **FAIL** (до внедрения required fixes).
- Ожидаемое состояние после внедрения required fixes и прохождения acceptance criteria: **PASS**.
